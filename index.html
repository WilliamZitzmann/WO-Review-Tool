Getting Started

§1
v0.23.0

WO Review Tool

A bookmarklet that injects a live checklist panel into Maximo work order pages — it reads field values and tables straight off the page, runs your own rules against them, and helps you write a clean return message in seconds instead of re-checking the same things by eye every time.

Installing the Bookmarklet

Drag the button below to your bookmarks bar. You install it exactly once.

Drag to your bookmarks bar
Review Tool
Doesn't drag easily, or need the raw code to paste into a manually-created bookmark instead? See the install page.
1
Open a work order in Maximo.
2
Click the bookmarklet. If you're not already on the right Maximo page, it redirects you there first — click it again once you've landed and are logged in.
3
The tool checks whether you have permission to use it, then loads the panel on the right side of the page.
4
First time only: pick a starting configuration (a shared preset, or start blank) in the installer that appears.
Access required: if you haven't been granted access yet, the tool will tell you who to contact. This is automatic — you won't need to do anything yourself once you're approved.

Your First Scan

Right after install, most of the panel has nothing to show yet — group tiles, status badges, and the return message all depend on data the tool hasn't read yet. That's expected, not a bug.

1
Open the work order you want to review.
2
Press Scan.
3
Wait for it to finish — the status line and scan log show progress as it visits each configured tab.
4
Review the results in each group tile.
Different PC? Your configuration lives in this browser, not your Maximo account — see Export / Import & Backup for how to carry it to another computer.

Typical Workflow

§2

Once you're set up, day-to-day use is short:

1
Open a work order.
2
Click Scan.
3
Review any issues found — group tiles show pass/fail/warn status and any messages.
4
Correct issues in Maximo if necessary, then re-scan.
5
Return or Approve the work order.

Everything else in this guide — rules, groups, scans, variables — is about configuring what happens in steps 2–3, not changing this basic shape.

Header Controls

Right under the header, a status line shows what the tool is currently doing or last did — e.g. "Scan complete 10:34:21 AM". Below that, a small scrollable scan log lists each scan step's result:

Log EntryMeaning

Group Tiles

Each group you configure shows up as a card. Click its header to expand or collapse it. Every tile can have:

Status badge
Summarizes every rule the group references. Hover it for the full breakdown.
Header message
An optional inline message next to the title — a field value, a variable's value, or a rule's pass/fail message. Configured per group in Setup.
Field rows
Captured field values, arranged however you laid them out in Setup.
Linked table
If the group has one, captured table rows appear inline with a column-visibility toggle.
Rule status blocks
Each rule the group references shows its status (✓ Pass / ✗ Fail / ⚠ Warn / — N/A) plus any configured messages underneath.

Drag a group by its header to reorder it. Use the hide (eye) icon on a group's header in Setup's Groups tab to hide it from the panel entirely — a "Show hidden tiles" button at the bottom of the panel brings it back.

Return / Approve

At the bottom, a preview box shows the assembled return message (built automatically from every failed/warned rule that's configured to contribute to it). You can edit this box directly — whatever's in it when you click Return or Copy (or use their hotkeys) is exactly what gets used, not a value recomputed behind your back. Re-scanning replaces it with a freshly built message again.

↩ Return
Opens Maximo's Route Workflow dialog, selects Return, and fills the Memo field with the message currently in the box. Asks you to confirm first.
✓ Approve
Opens Route Workflow, selects Complete Review. Asks you to confirm first.

Both can be assigned their own keyboard shortcut — see Keyboard Shortcuts.

Adding Fields

§5

Before a field can be used in a rule or shown in a group, it has to be registered once. There are two ways, both in Setup → Groups & Display.

Browse Page Fields (recommended)

1
On the actual Maximo page (behind Setup), navigate to the tab that has the field you want.
2
In Setup → Groups & Display, click 🔍 Browse Page Fields.
3
It lists every labeled field it can currently see on the page (across every Maximo tab already loaded), with its current value. Already-registered fields are highlighted green.
4
Filter by typing, tick the fields you want, optionally pick a group to drop them straight into, then click Save Selected.
Only see fields from one tab? Maximo often only builds a tab's fields into the page once you've actually visited that tab. Click through the tabs you care about first, then open Browse Page Fields.

Add a Field Already Registered

If a field is already registered (from a shared preset, or registered previously) but isn't in any group yet, use the plain "Add a field not yet in any group" dropdown right above the Browse button — pick the field and a target group, click Add.

Once registered, a field is available everywhere as F('Tab :: Label') — in rules, variables, scan actions, and message text. You don't need to re-register it for each new rule.

Rules

§6

A rule is a small formula evaluated against whatever the last scan captured. It returns a status, which drives the panel's colors and the return message.

A Simple Rule, in Plain English

Say you want to make sure a field is filled in:

"Production Run Lot Number must not be blank."

That's the whole idea of a rule: check something about the work order, and decide pass or fail. In code, that becomes: read the field, then return true if it has a value or false if it doesn't. The next section shows exactly that rule written out.

Return Values

Return ValueStatusColor
truePass Green
falseFail Red
'warn'Warning Orange
'na'Not Applicable Yellow
(throws an exception)Error Purple

Writing a Rule

In Setup → Rules, click + Add Rule. Give it a label, then write its formula — plain JavaScript, using return to give your answer. Here's the lot-number example above, written out:

var lot = F('Work Order :: Production Run Lot #');
if (!lot) return 'na';
return lot.length > 3;

Every field you've registered (see Adding Fields) is available through F('Tab :: Label'). The full helper-function list is in the Formula Reference.

Pass / Fail / Warn Messages

Each rule has three collapsible sections — Pass, Fail, Warn — all with the identical shape:

Short
One line shown next to the group title (or wherever the group's header message points at this rule). Supports {{expr}}.
Long
A bullet list shown inside the expanded tile. Each entry has its own Condition (optional formula — blank means always show it) and Message text. Add more with "+ Add message".
Return Mode
What this status contributes to the return message: none, short (the Short text), long (every matching Long entry), or custom (its own text below). Pass has no return mode — only Fail/Warn feed the return message.

{{expr}} Interpolation

Drop a live value into any message (Short, a Long entry, or a custom return message) with {{...}}:

"Lot: {{F('Lot Number')}}"

It's evaluated as a single expression — don't use return inside it. For anything that needs more than one step, wrap it in a function and call it immediately:

"Hours: {{(function(){ var h=hoursBetween(F('Actual Start'),F('Actual Finish')); return h?h.toFixed(1):'?'; })()}}"
Seeing a ? instead of a value? The expression threw an error — double-check the field name you're passing to F().

More Code Examples

Duration vs. actual hours

var d = hours(F('Duration'));
var a = hoursBetween(F('Actual Start'), F('Actual Finish'));
if (!d || !a) return 'na';
return d <= a;

Warn on emergency work type

var wt = F('Work Type');
if (wt === 'EM') return 'warn';
return true;

Table row-count check

return rowCount('Labor') > 0;

Using a variable

var total = parseFloat(V('total_labor_hours'));
if (isNaN(total)) return 'na';
return total >= 0.5;

Variables

§7

Think of variables as reusable calculations that can be shared between multiple rules — instead of writing the same formula in three different rules, write it once as a variable and reference it from each one.

A variable is a named formula that returns any kind of value (a string, a number, whatever) instead of a pass/fail status. Variables are computed after each scan, before rules run, so rules — and other variables defined later — can use them.

Creating One

In Setup → Variables, give it an ID (used in code, e.g. v_total_hours), a Label (shown in the UI, also usable in V()), and a Formula — the same environment as a rule, just returning any value instead of a status.

Using a Variable

WhereHow
Inside a rule/scan/action formulaV('v_total_hours') or V('Total Hours') (label also works)
As a group's header messageSet the group's header message type to Variable in Setup → Groups
As a field row in a groupAdd it to the group's Variable Fields list

Example: work type + status summary

return F('Work Order :: Work Type') + ' — ' + F('Work Order :: Status');

Example: total labor hours from a table

var rows = T('Labor');
var total = 0;
rows.forEach(function(r) {
  total += hours(r['Regular Hours'] || '0');
});
return total.toFixed(2) + ' hrs';
Order matters: variables compute top-to-bottom. One can reference an earlier variable via V(), but not one defined below it.

Groups & Display

§8

A group is one tile card on the panel. Click + Add Group in Setup → Groups & Display to start one, then build it up:

What a Group Can Show

ContentDescription
Field RowsRegistered fields, arranged in rows — group several per row for a compact layout, or one per row for long values
Variable FieldsVariable values, shown alongside your other fields
Linked TableA captured table's rows, inline, with a column-visibility toggle
RulesTick which rules this group references — their status blocks appear inside, and their statuses drive the group's badge

Header Message

Turn on "Show inline header message" to put a short line next to the group's title. Three types:

Field value
Shows a registered field's captured value.
Rule pass/fail message
Shows a rule's Short message, colored by its status. Leave the rule dropdown on Auto and it automatically shows whichever of the group's own rules currently needs the most attention — pick a specific rule instead if you always want that one rule's message regardless of the others.

Tooltip & Expanded Message

A group can also have a small ⓘ tooltip icon (static text, shown on hover) and an Expanded Message — a banner shown at the top of the tile only while it's expanded.

Collapsed by Default, Visibility, Reordering

  • Use the eye icon to hide a group from the panel entirely — hiding also collapses it; showing it again leaves its expand state alone.
  • Drag a card by its handle (left edge) to reorder — the same order is used on the panel.

Scan System

§9

A scan visits every configured target in order — clicking a Maximo tab or opening a dialog, waiting for it to load, capturing whatever it needs, then moving to the next one — and finishes back on the WO tab. Configure targets in Setup → Scan.

Scan Target Properties

Type
Tab — clicks a Maximo tab link. Dialog — triggers a popup dialog (e.g. Route Workflow) instead.
Tab ID / Event
For a Tab target, the internal Maximo element ID of the tab link to click. For a Dialog target, the trigger event type. See "Finding IDs" below.
Wait for text / Wait for table
What tells the tool this target has actually finished loading — either a marker text string that should appear, or the internal name of a table that should be populated. Set one, not both.
Condition
Optional formula — return true to scan this target this run, false to skip it. Leave blank to always scan it. Conditions can see data captured by earlier targets in the same run.
Row Detail Fields
For a target with an expandable table: fields captured from inside each row's own expanded detail panel, not the table's visible columns.
Post-Scan Actions
Auto-fill Maximo fields once this target's data is captured — see below.

Finding a Tab ID or Field Element ID

These are raw Maximo DOM element IDs, not anything the tool can look up for you by name. To find one:

1
Right-click the tab or field in Maximo and choose Inspect (Chrome/Edge DevTools).
2
Look at the highlighted element's id attribute — Maximo tab links typically end in -tab, and field inputs typically end in -tb or similar.
3
Copy that ID into the Tab ID / Field Element ID box.
These IDs can change between Maximo versions/configurations. If a scan target or action suddenly stops working after a Maximo update, this is the first thing to re-check.

Post-Scan Actions requires the Fix beta feature

An action fills one Maximo field automatically once its target has been scanned. This whole feature is currently gated behind the Fix beta feature (see Beta Features) since it's still being tested — if you don't have that enabled, the editor is hidden but any actions you'd already configured are kept, untouched, ready for when you turn it on.

Field Element ID
The Maximo field to fill (found the same way as a Tab ID, above).
Value Expression
What to put there — a variable (V('v_core')) or a formula (F('...') and friends). Evaluated fresh every time the action runs.
Condition
Optional formula — leave blank to always run this action once its target is scanned.

Table & Dialog Capture

When a scan target's Wait Table (or a group's Linked Table) names a table, the tool reads its rows directly off the page, available in formulas via T('Table Name'), rowCount(...), col(...), has(...), and lookup(...) (see the Formula Reference). A Dialog-type target opens a Maximo popup instead of switching tabs, captures data the same way, then dismisses it automatically — you won't see it stay open.

Setup's Tables tab lists every table your Groups/Scan config references and lets you give it a friendlier display name. It also lets you build a custom table — small hand-entered lookup data that didn't come from a Maximo scan (e.g. part numbers, cost centers) — which formulas can read with the exact same T()/col()/has()/lookup() functions as a scanned table.

The Tables tab also has an API Tables section requires the Maximo REST Data beta feature — a table backed by a live REST lookup instead of a scan or hand-typed rows: recent work orders or downtime history for an asset. You give it a Source, and formulas for which asset/site to pull (so the same table definition can follow whichever WO you're currently reviewing). Once set up, it reads through T()/lookup()/col() exactly like any other table — see Beta Features.

Profiles & Presets

§10

A profile is a named, portable snapshot of your rules/groups/scan/variables (not device settings like hotkeys or your update channel — those stay with the device). Useful for keeping a couple of different setups around, or sharing yours with a colleague.

Local Profiles

  • Save Current As New Profile — snapshots what you have right now under a name you choose.
  • Switch — swaps your live configuration to a saved profile (blocked if it's already active, or your only one).
  • Start Blank Profile — wipes the current configuration to a fresh empty one.
  • Delete — removes a saved profile (blocked for the active one, or your last remaining one).

Shared Presets

The Profiles tab can also list ready-made presets others have published. Installing one downloads and registers it as a local profile, then switches you to it — your previous configuration isn't lost, just no longer active (switch back any time). See Export / Import & Backup for how to publish your own.

Settings & Hotkeys

§11

Everything in Setup → Settings is device-level — it stays with this browser, and isn't affected by switching profiles.

Hotkeys

Scan, Return, Approve, Copy Return Message, and (if enabled) Fix each have their own keyboard shortcut, all grouped under one Hotkeys section. Click a box, press the key combination you want, and it's saved immediately (no Save & Apply needed). The same combination can never be assigned to two actions at once. Hotkeys never fire while you're typing in one of the tool's own text fields. See Keyboard Shortcuts for the current defaults.

Auto-Scan

When enabled, opening a different work order than the last one you scanned triggers a scan automatically, without needing to click Scan yourself.

Quick Return Message

Set a Prefix (e.g. "Hi {name},"), Suffix/Signature (e.g. "- wz"), and the delimiter used between combined messages — these wrap around whatever your rules contribute to the return message. If your rules produce no message, the prefix/suffix stay out of it too, rather than showing on their own.

Display

Toggle whether the pass/fail/warn/error count summary shows under the status line.

Update Channel

Most people don't need to touch this — updates apply automatically. If you ever need to pin to a specific version or roll one back, that option lives here too.

Reset / Uninstall

Two options if you want a clean slate:

Reset Tool
Clears cached tool data but keeps your rules/groups/scans/variables — they're restored automatically the next time you click the bookmarklet.
Full Reset
Erases everything, with no way back. Use this to start completely fresh.

Neither one is a permanent uninstall — clicking the bookmarklet again always reinstalls the tool, since there's no way to remove it from a page you don't control.

Export / Import & Backup

§13

Your rules, groups, scan targets, variables, and field registrations live in this browser's local storage — not in your Maximo account, and not automatically shared between computers or browsers. There are three ways to move or protect that configuration:

Export / Import
Setup → Export produces a backup file of your full configuration; Import restores from one. Use this to move to a different PC, keep a manual backup, or send your setup to someone else.
Auto-Backup
Setup → Settings — on Chrome/Edge, link a local file once and the tool keeps it updated automatically every time you save a change. The most reliable option if your browser supports it.
Shared Presets
If you want to share your setup with a whole team rather than one person, ask whoever maintains this tool to add your exported configuration to the shared preset list — see Profiles & Presets.
Set up at least one of these — without it, losing the browser profile (a new PC, a reset browser, a cleared history) means starting your configuration over from scratch.

Feedback

§14

Setup → Feedback is the fastest way to report a bug or suggest an improvement. Pick a type, describe what happened (or what would help), and click Send — it's filed directly, along with your tool version, current channel, and browser info, so nothing needs to be typed out by hand. If it can't be filed for any reason, it falls back to opening an email draft instead, so a report is never just lost.

Beta Features

§15
Most people won't see this tab at all. If you haven't been granted beta access, skip this section — nothing here applies.

Beta features are optional and off by default even if you're granted access to them — you choose whether to turn each one on in the Beta tab. They may change, or be removed, as they're refined, so don't build critical workflows around one without expecting it to move.

Currently available:

Route / Fix
Adds a Route symbol and a yellow Fix button next to Return/Approve, plus Post-Scan Actions (see Scan System). Fix re-runs a scan and reapplies any actions marked for it — a quick way to correct a field without a full Return/Approve round trip.
Maximo REST Data (experimental)
Three formula helpers - domain(key, code), assetWOHistory(assetnum, siteid, limit), and assetDowntimeHistory(assetnum, siteid) (see Formula Reference) - plus API Tables in the Tables tab (see Scan System), which wraps the same asset work order / downtime history lookups into a named table you can reference with T()/lookup()/col(). The asset ones pull live from Maximo's own REST API rather than anything scanned/scraped, and are genuinely experimental - data shapes and field reliability are still being verified. Don't build a rule you depend on around these yet.

FAQ

§16
Does the tool modify Maximo?
Only when you tell it to — clicking Return, Approve, or (beta) Fix. Scanning only reads data; it never writes anything on its own.
Can I share my rules?
Yes — export a backup file and send it, or ask whoever maintains this tool to publish it as a shared preset. See Export / Import & Backup.
Why is a field blank?
Either it isn't registered yet (see Adding Fields), or the Maximo tab it lives on hasn't been visited/scanned this session.
Why didn't Scan find a field?
The tab it's on may not have loaded in time, its element ID may have changed (Maximo updates can do this), or its scan target's "wait for" marker never appeared.
Does configuration move with me to another computer?
Not automatically — it's stored in this browser. Use Export/Import or Auto-Backup to carry it over; see Export / Import & Backup.

These helper functions are available inside every rule, variable, scan condition, and post-scan action expression. This same list is searchable inside the tool itself — click the 📖 button next to Save in Setup — so you don't need to keep this page open in another tab while you write a formula. Typing a function's name into any formula box also suggests matching functions as you type, even before you've opened its parenthesis.
